# 优题慧家长系统 - 后端架构与方案设计

## 1. 技术栈与架构
- **核心框架**: Spring Boot 3.x
- **开发语言**: Java 21
- **安全控制**: Spring Security + JWT
- **数据库**: MySQL 8.0 (持久化存储)
- **缓存与消息队列**: Redis (Token存储、短信验证码、简单的异步任务队列)
- **文件存储**: 阿里云 OSS / 腾讯云 OSS / MinIO (用于存储试卷、PDF、Word 报告)
- **持久层框架**: MyBatis-Plus 或 JPA

## 2. 核心模块与接口设计

### 2.1 认证与权限服务 (Auth Service)
- **接口**:
  - `POST /api/auth/send-sms`: 接收手机号，生成随机验证码存入 Redis（设置5分钟过期），并调用短信网关发送。
  - `POST /api/auth/login-sms`: 校验手机号和验证码，成功后下发 JWT Token。
  - `POST /api/auth/login-wechat`: 接收 OpenID，如果已绑定手机号则直接返回 Token，否则返回特定状态码引导绑定手机。
- **安全拦截**:
  - `JwtAuthenticationFilter`: 解析请求头中的 Token，解析出用户角色（家长/后台管理员/VIP状态），并存入 SecurityContext。

### 2.2 学生与家长关系服务 (Student-Parent Service)
- **数据库设计**:
  - `Parent`: 手机号、第三方 OpenID、VIP类型 (普通/VIP/SVIP)、VIP到期时间。
  - `Student`: 学号、姓名、学校、年级、班级。
  - `Parent_Student_Relation`: 家长与学生的多对多关联表。
- **核心逻辑**:
  - 绑定学生时，后端通过模糊查询学校库和学生表进行信息比对，必须姓名、学校、年级完全匹配才允许建立关联。

### 2.3 核心业务：试卷与成绩解析服务 (Core Data Service)
- **业务痛点解决**: 上传 Word/PDF 格式试卷和答案并自动识别知识点。
- **方案设计**:
  1. **OCR与版面分析**: 集成如 阿里/腾讯 OCR 或本地部署开源的 PaddleOCR 进行试卷 PDF/图片的文本提取。
  2. **NLP 与知识点提取**: 
     - 将提取的题目文本发送给 AI 引擎（如调用大模型 API 或基于特定学科的 NLP 模型）。
     - 提示词设计（Prompt）：要求大模型分析题目文本并返回所属的知识点标签。
  3. **数据结构化**: 将解析后的试卷结构（题号、知识点、正确答案）落库。

### 2.4 数据导入与批处理服务 (Batch Task Service)
- **Excel/ZIP 解析**:
  - 采用 EasyExcel 快速读取学生基础信息表、小题分文件包。
  - 对于上传的试卷压缩包，后端异步解压并推送到 OSS，记录路径。
- **成绩分析引擎**:
  - 根据导入的小题分结合题目关联的知识点，后台定时任务（或 MQ 消费者）自动聚合生成成绩构成、分布、趋势数据。
- **错题推荐与 PDF 生成 (SVIP 举一反三)**:
  - **推荐逻辑**: 找出学生得分率为 0 的小题 -> 查询对应的知识点 -> 在题库中随机抽取相同知识点的 3 道同类型题目。
  - **文件生成**: 采用 Apache PDFBox 或 iText (对于 Word) 动态生成包含错题解析及举一反三推荐题的文档，生成后上传 OSS，将链接返回给小程序端下载。

### 2.5 AI 课程与计划服务 (AI Service)
- **接口设计**:
  - `GET /api/ai/plan`: 基于该学生的弱点知识点库，组装 Prompt 请求大模型生成个性化的学习日程安排。
  - `GET /api/course/public`: 查库返回上架的免费视频课程列表。
  
### 2.6 接口规范与数据安全
- **统一返回格式**:
  ```json
  {
      "code": 200,
      "msg": "操作成功",
      "data": {}
  }
  ```
- **数据隔离**: 在数据查询层（MyBatis 插件或 AOP）植入当前登录家长的关联学生 ID 限制，防止水平越权（例如查询别人家孩子的成绩）。
- **加密策略**: 数据库中的密码等敏感字段使用 BCrypt 加密，传输层强制采用 HTTPS。
